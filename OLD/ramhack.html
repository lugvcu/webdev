RamHack

When: Saturday May 7th 2011 1:00PM

Where: Harris Hall Room 3101


Summary:

The Linux User Group of VCU is sponsoring an open hacking event called RanHack. This event is focused on penetration testing and computer security awareness. Everyone is welcomed to attend regardless of experience level. There will be opportunity for those interested in learning about hacking and computer security as well as opportunity for people to try out their skills at breaking into our servers. Individuals as well as teams are welcome to attend. No registration is needed and there are no restrictions on participants.

NOTE: This event will be held on a wired isolated network built specifically for this event. There will be no connection to the VCU network and attendees should disable wireless adapters to prevent accidentally spawning attacks on unintended networks.

Rules:

    * Users must be connected only to the test network set up by the Linux User Group. If there are any questions we will be happy to help ensure your network connections.
    * Don't attack the routers/switches or network connection gear.
    * Don't attack other users participating in the event.
    * Don't change the server configuration once you gain access (ie. securing the server so others can't gain access.)
    * Bring an Ethernet cable at minimum, bringing extension cables for power and an extra network switch is recommended.
    * If you plan on participating as a team, the team will be given one Ethernet port. The team is responsible for bringing any additional network gear needed to serve connections to the rest of the team. There are no restrictions on team size or equipment used.


Objectives:

There will be multiple servers on the network and possible a client machine or two. The IP addresses of these machines will be given out so there is no doubt which machines will be targeted during the competition. There will be an "unsecured" machine and a "secured" machine to facilitate hacking for attendees of various experience levels. The goal will be to gain access to the machines and obtain information stored on the machines.

In order for your exploits to be recognized, each person/team should record the flowing information:

    * The machine that was accessed
    * The method or exploit used
    * The result or possible significants of the access gained
    * Any information found as a result of the "hack"


Disclaimer:

The Linux User Group supports ethical hacking and it's use in perpetuating best security practices. We do not condone or tolerate any illegal activity. We intend this even to be a fun and educational experience by providing a save and secure environment to explore hacking practices and it's application to security penetration testing. Even though it is specifically forbidden to attack other attendees computers and we see no reason to suspect this would be a problem, the Linux User Group and VCU is not responsible for any equipment or data affected by the competition. To be on the safe side, all important information should be backed up before the competition. Ideally a dedicated machine could be used at the event to prevent the possibility of data loss. Please contact us if there are any questions.
